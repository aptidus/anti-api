Available Models (Flow Routes)

Spear Agents uses flow route model IDs — semantic names without version numbers. Each route maps to upstream provider chains with automatic failover and load balancing.

OpenClaw Setup (Step-by-Step)

Follow these steps to connect OpenClaw to Spear Agents:

1Install OpenClaw (if not already installed) 
npm install -g openclaw
2Open your config file 
nano ~/.openclaw/openclaw.json

If the file doesn't exist yet, create it. If it already has content, merge the following into your existing config.

3Add the Spear Agents provider

Paste this into your openclaw.json. Replace ak-YOUR_KEY with your actual API key:

{
  "models": {
    "mode": "merge",
    "providers": {
      "spear-agents": {
        "baseUrl": "https://anti-api-proxy-production.up.railway.app/v1",
        "apiKey": "ak-YOUR_KEY",
        "api": "openai-completions",
        "models": [
          { "id": "Opus-thinking", "name": "Opus Thinking", "reasoning": true },
          { "id": "Sonnet-thinking", "name": "Sonnet Thinking", "reasoning": true },
          { "id": "Haiku", "name": "Haiku" },
          { "id": "Codex", "name": "Codex" },
          { "id": "Codex-Mini", "name": "Codex Mini" }
        ]
      }
    }
  },
  "agents": {
    "defaults": {
      "model": {
        "primary": "spear-agents/Opus-thinking"
      }
    }
  }
}
4Restart the gateway 
openclaw gateway restart
5Verify it works 
openclaw agent --local -m "Hello, what model are you?" --session-id test --json

You should see "provider": "spear-agents" in the output.

Tools work natively. OpenClaw's built-in tools (file read/write, web search, process, etc.) work out of the box with all Spear Agents models — no extra configuration needed.

Claude Code Setup (Step-by-Step)

Follow these steps to connect Claude Code to Spear Agents:

1Install Claude Code (if not already installed) 
npm install -g @anthropic-ai/claude-code
2Set environment variables

Add these to your shell config (~/.zshrc or ~/.bashrc):

export ANTHROPIC_BASE_URL="https://anti-api-proxy-production.up.railway.app"
export ANTHROPIC_API_KEY="ak-YOUR_KEY"

Then reload your shell:

source ~/.zshrc
3Set your preferred model 
claude config set --global model Opus-thinking

Or use any flow route model ID from the table above.

4Test it 
claude -p "Hello, what model are you?"

You should receive a response from the model.

5Test with tools 
claude -p "Read the file package.json and tell me the project name"

Claude Code should use the file-read tool and return the project details.

All Claude Code tools work. File editing, bash commands, web search — everything works through the proxy exactly as with a native Anthropic API key.
Note: Do NOT set ANTHROPIC_BASE_URL with a trailing /v1. Claude Code adds its own path. The correct value is the root URL without /v1.

Extended Thinking

Models with -thinking suffix (e.g. Opus-thinking, Sonnet-thinking) automatically enable extended thinking / chain-of-thought reasoning. The proxy handles this transparently:

  • OpenAI format (/v1/chat/completions): Thinking output is returned in reasoning_content field on each choice/delta
  • Anthropic format (/v1/messages): Thinking blocks are returned natively as thinking content blocks
  • Budget tokens are automatically calculated based on max_tokens (minimum 1024)

No extra parameters needed — just use a -thinking model ID and thinking is enabled automatically.

扩展思考

后缀为 -thinking 的模型（如 Opus-thinking、Sonnet-thinking）会自动启用扩展思考/链式推理。代理透明处理：

  • OpenAI 格式（/v1/chat/completions）：思考输出通过 reasoning_content 字段返回
  • Anthropic 格式（/v1/messages）：思考块作为 thinking 内容块原生返回
  • 预算 token 根据 max_tokens 自动计算（最小 1024）

无需额外参数 — 只需使用 -thinking 模型 ID 即可自动启用。

Agentic Mode

Spear Agents fully supports agentic workflows — multi-turn tool use loops where the model calls tools, receives results, and continues reasoning until the task is complete. This is the mode used by Claude Code, OpenClaw, and other AI coding agents.

How It Works

  • Multi-turn tool loops: The model returns stop_reason: "tool_use" (Anthropic) or finish_reason: "tool_calls" (OpenAI) when it wants to call a tool. Your client executes the tool and sends the result back. This loop continues until the model returns end_turn / stop.
  • Interleaved thinking: When using -thinking models, the model can think between tool calls — reasoning about results before deciding the next action. The proxy supports the interleaved-thinking beta for streaming thinking blocks alongside tool use.
  • Streaming tool calls: Tool call arguments stream incrementally via fine-grained-tool-streaming, so your client can start parsing tool inputs before the full call is complete.
  • Automatic format translation: Send tool results in either OpenAI format (tool role messages) or Anthropic format (tool_result content blocks) — the proxy translates between them transparently.

Agent 模式

Spear Agents 完全支持 Agent 工作流 — 多轮工具调用循环，模型调用工具、接收结果、继续推理直到任务完成。这是 Claude Code、OpenClaw 等 AI 编程 Agent 使用的模式。

工作原理

  • 多轮工具循环：模型返回 stop_reason: "tool_use"（Anthropic）或 finish_reason: "tool_calls"（OpenAI）时表示需要调用工具。客户端执行工具并返回结果，循环继续直到 end_turn / stop。
  • 交错思考：使用 -thinking 模型时，模型可以在工具调用之间思考 — 分析结果后再决定下一步操作。
  • 流式工具调用：工具调用参数通过 fine-grained-tool-streaming 增量流式传输，客户端可以在完整调用完成前开始解析。
  • 自动格式转换：可用 OpenAI 格式（tool 角色消息）或 Anthropic 格式（tool_result 内容块）发送工具结果 — 代理自动转换。
